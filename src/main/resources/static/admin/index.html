遇到问题再展开 FAQ

FAQ 默认收起来，只在你卡住时再看“第一步做什么”“为什么导入后还不能用”“什么时候该去别的页面”。

第一次进入这个页面先做什么

先看首屏状态判断当前能不能直接问；如果还不能问，就先去“资料导入”上传文件或接 Git 仓库；导入后切到“当前处理任务”看处理进度，最后再到“已入库内容”确认结果。

为什么上传后一直看不到入库结果

先到“当前处理任务”看任务是不是还在排队或运行中；如果已经失败，优先看资料源与文件区块里的同步历史和文件解析方式，再决定是资料格式问题、解析问题，还是连接问题。

Git 私有仓库导入失败怎么办

先确认仓库类型是不是切到了“私有仓库”，并且已经选中可用凭据；如果凭据还没有，直接在当前页面新增。公开仓库不需要凭据，私有仓库最常见的问题就是忘了切仓库类型或凭据失效。

为什么资料看起来导进来了，问答还是答不出来

先在“已入库内容”里搜索关键标题、摘要或概念词，确认内容是否真的入库；如果这里都搜不到，就说明还没进入可问答状态。只有资料真正进入知识库，问答页才更容易命中。

什么时候该去系统配置

普通导入、同步和查看结果都留在工作台；只有在需要改模型与向量、OCR、服务器目录资料源时才去“系统配置”。如果你要接 CLI、HTTP API 或 MCP，再去“开发接入”。
