- 当前 `t1.md` 提供的 OpenAI 兼容网关能处理 chat，但不支持当前 Spring AI OpenAI EmbeddingModel 所调用的 embeddings 接口

影响：

- 向量索引会 fail-open
- Query 仍可通过 FTS / SOURCE / ARTICLE 回答
- 但本轮 **不能宣称向量检索已在真实网关下生效**

后续收口补充：

- admin 已新增 `GET /api/v1/admin/vector/config` 与 `PUT /api/v1/admin/vector/config`
- admin 已新增 `GET /api/v1/admin/vector/status` 与 `POST /api/v1/admin/vector/rebuild`
- 后台页已支持保存向量开关、embedding 模型、期望维度，并查看向量启用状态、schema 维度、已索引文章数、最近更新时间
- 向量配置保存后会立即作用到运行时 embeddings 请求，而不再只是展示层标签
- 向量重建跳过条件已从“只看 `contentHash`”收口为“同时校验 `contentHash + modelName`”
- 当启用向量检索或修改 embedding 模型 / 维度时，后台会返回建议继续执行重建向量索引的提示
