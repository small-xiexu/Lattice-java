本清单是后续实施的唯一进度台账。执行时按阶段推进，每完成一个可独立验收事项，必须立即回写状态和验收依据。

- `未开始`：尚未动工

## 2. Evidence Card 生成

### 2.1 生成入口

- [ ] 确定 compile 产卡节点
  - 状态：未开始
  - 候选位置：`compiler/service` 或 `compiler/graph/node`
  - 要求：在 source chunk 持久化后、article 生成后均可读取上下文，但精确事实以 source chunk 为准。
  - 验收：compile 成功后能生成 fact card，不影响 article 落库。
- [ ] 新增 `FactCardGenerationService`
  - 状态：未开始
  - 建议位置：`src/main/java/com/xbk/lattice/compiler/service/FactCardGenerationService.java`
  - 输入：`source_file_chunks`、`AnalyzedConcept.sections`、必要 article 背景。
  - 输出：`FactCardRecord` 列表。
  - 验收：可在无 LLM 场景下用规则生成前三类卡。

### 2.2 结构块识别

- [ ] 实现通用列表块识别
  - 状态：未开始
  - 产物：`FACT_ENUM`
  - 规则：bullet、numbered list、表格单列/多行、重复键值结构。
  - 验收：不少于 5 个通用样例单测，不包含业务专名。
- [ ] 实现通用对照块识别
  - 状态：未开始
  - 产物：`FACT_COMPARE`
  - 规则：双列以上表格、`A/B` 对照、旧/新、现状/目标、配置/实现差异等通用结构。
  - 验收：对照卡必须至少包含双方字段；缺一侧进入 `incomplete`。
- [ ] 实现通用顺序块识别
  - 状态：未开始
  - 产物：`FACT_SEQUENCE`
  - 规则：阿拉伯数字序号、中文序号、阶段/批次/步骤连续结构。
  - 验收：顺序号保留，query 和回答不得打乱顺序。
- [ ] 实现状态块识别
  - 状态：未开始
  - 产物：`FACT_STATUS`
  - 规则：已确认、待确认、删除、移除、改为、无流量、有流量等通用状态分组。
  - 验收：同一事项不可同时落入互斥状态，冲突时进入 `conflict`。
- [ ] 实现规则块识别
  - 状态：未开始
  - 产物：`FACT_POLICY`
  - 规则：必须、不得、原则、统一要求、强约束、禁止项、适用边界。
  - 验收：卡片必须包含约束和适用范围，缺边界时进入 `incomplete`。

### 2.3 source 回指校验

- [ ] 实现 `evidenceText` 定位校验
  - 状态：未开始
  - 要求：高置信卡的 `evidenceText` 必须能在 `source_file_chunks.chunk_text` 中定位。
  - 验收：无法定位的字段不能进入 `valid`。
- [ ] 实现 source 邻接窗口合并
  - 状态：未开始
  - 目标：处理表格跨 chunk、连续编号跨 chunk、标题与列表分离。
  - 验收：跨 chunk 清单不会被拆成多张互相缺项的卡。
- [ ] 实现 `contentHash` 去重
  - 状态：未开始
  - 目标：同一 source 结构块重复 compile 时不重复落卡。
  - 验收：重复编译同一文件后 fact card 数量稳定。

### 2.4 生成测试

- [ ] 新增 `FactCardGenerationServiceTests`
  - 状态：未开始
  - 覆盖：枚举、对照、顺序、状态、规则、source 回指失败、去重。
  - 验收：测试样例不得包含卡券文档业务专名。
- [ ] 新增真实 source chunk 小样本测试
  - 状态：未开始
  - 目标：用通用复杂文档片段验证表格、列表、状态、顺序。
  - 验收：每类至少 1 个真实片段。
