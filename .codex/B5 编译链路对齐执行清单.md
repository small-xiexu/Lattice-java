# B5 编译链路对齐执行清单

> **唯一进度台账**：自本文件创建起，B5“编译链路对齐”阶段的执行进度、进行中事项、阻塞信息、验收依据统一只回写到本文件，不再分散记录到其他文档。

**目标**：让 Java 版从“最小编译闭环”演进为接近原始 Lattice 的知识编译器，补齐多格式采集、LLM 分析、Markdown 文章编译、单轮审查修复、增量编译与合成产物。

**范围边界**：

- 本清单只覆盖 B5，不包含 B6 查询反馈闭环、B7 治理/MCP 全工具集、B8 Java 增强项
- 本阶段不引入 OSS
- 本阶段不把 `Agent Framework` 作为前置
- 本阶段不把 `pg_jieba` / `pgvector` 作为闭环前提

**设计约束**：

- 原始文件统一走本地文件系统
- PDF / Excel / 图片先在采集层转成规范化文本/占位信息，再进入统一编译链
- `articles.content` 必须升级为完整 Markdown（含 YAML frontmatter）
- 审查流程保持与原版一致：单轮审查 + 单次修复
- B5 所有完成勾选必须有代码或测试依据
