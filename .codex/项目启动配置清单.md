# Lattice Java 项目启动配置清单

**文档目的**：用于本项目本地启动前的配置核对与启动验收。  
**适用范围**：当前仓库开发环境，基于 `Spring Boot 3.5.1 + JDK 21 + PostgreSQL 16 + Redis 7`。  
**配置基线来源**：以 `pom.xml`、`docker-compose.yml`、`src/main/resources/application.yml`、`src/main/resources/db/migration/*` 为准。

---

## 1. 启动前环境核对

- [ ] 当前 `java` 指向 **JDK 21**，执行 `java -version` 与 `mvn -version` 确认不是 Java 8
- [ ] Maven 可正常使用，默认沿用全局本地仓库：`/Users/sxie/maven/repository`
- [ ] 若全局 Maven 镜像再次出现 `alimaven` 握手不稳定，可临时改用：`.codex/maven-settings.xml`

- [ ] 本机端口 `5432`、`6379`、`8080` 未被其他进程占用，或已确认要复用现有服务

建议核对命令：

```bash

lsof -i :6379

```

## 3. 应用启动配置核对

### 3.1 最小本地启动结论

当前 `application.yml` 已为本地开发提供默认值。若你直接使用仓库自带的 `docker-compose.yml`，并保持默认端口与账号密码不变，则应用**理论上无需额外环境变量也可启动**。

但为了避免环境串线，仍建议显式设置以下变量。

### 3.2 建议显式设置的变量

- [ ] `SPRING_PROFILES_ACTIVE=jdbc`
- [ ] `SPRING_DATASOURCE_URL=jdbc:postgresql://127.0.0.1:5432/ai-rag-knowledge?currentSchema=lattice`
- [ ] `SPRING_DATASOURCE_USERNAME=postgres`
- [ ] `SPRING_DATASOURCE_PASSWORD=postgres`
- [ ] `SPRING_FLYWAY_ENABLED=true`
- [ ] `SPRING_FLYWAY_SCHEMAS=lattice`
- [ ] `SPRING_FLYWAY_DEFAULT_SCHEMA=lattice`
- [ ] `SPRING_AI_OPENAI_API_KEY` 已设置为任意非空值
- [ ] `SPRING_AI_ANTHROPIC_API_KEY` 已设置为任意非空值
- [ ] `LATTICE_REDIS_HOST=127.0.0.1`
- [ ] `LATTICE_REDIS_PORT=6379`

建议导出命令：

```bash
export SPRING_PROFILES_ACTIVE=jdbc
export SPRING_DATASOURCE_URL="jdbc:postgresql://127.0.0.1:5432/ai-rag-knowledge?currentSchema=lattice"
export SPRING_DATASOURCE_USERNAME=postgres
export SPRING_DATASOURCE_PASSWORD=postgres
export SPRING_FLYWAY_ENABLED=true
export SPRING_FLYWAY_SCHEMAS=lattice
export SPRING_FLYWAY_DEFAULT_SCHEMA=lattice
export SPRING_AI_OPENAI_API_KEY=any-non-empty-value
export SPRING_AI_ANTHROPIC_API_KEY=any-non-empty-value
export LATTICE_REDIS_HOST=127.0.0.1
export LATTICE_REDIS_PORT=6379
```

### 3.3 可选增强配置

- [ ] 若需显式控制编译模型，设置 `LATTICE_LLM_COMPILE_MODEL`，默认 `openai`
- [ ] 若需显式控制审查模型，设置 `LATTICE_LLM_REVIEWER_MODEL`，默认 `anthropic`
- [ ] 若需开启 LLM 审查，设置 `LATTICE_LLM_REVIEW_ENABLED=true`，默认 `false`
- [ ] 若需调整 LLM 预算，设置 `LATTICE_LLM_BUDGET_USD`，默认 `10.0`
- [ ] 若需调整编译 WAL TTL，设置 `LATTICE_COMPILER_WAL_TTL_SECONDS`，默认 `86400`
- [ ] 若需调整查询缓存 TTL，设置 `LATTICE_QUERY_CACHE_TTL_SECONDS`，默认 `3600`
- [ ] 若需关闭增强 FTS 探测，设置 `LATTICE_QUERY_SEARCH_FTS_ENABLED=false`
- [ ] 若需指定首选 FTS 配置，设置 `LATTICE_QUERY_SEARCH_FTS_CONFIG`，默认 `jiebacfg`
- [ ] 若需指定 FTS 回退配置，设置 `LATTICE_QUERY_SEARCH_FTS_FALLBACK`，默认 `simple`
- [ ] 若需开启向量检索，设置 `LATTICE_QUERY_SEARCH_VECTOR_ENABLED=true`
- [ ] 若开启向量检索，确认模型与维度仍为 `text-embedding-3-small + 1536`

补充说明：

- 当前 FTS 首选配置为 `jiebacfg`
- 若数据库中不存在该 `tsconfig`，系统会自动回退到 `simple`
- 当前向量检索默认关闭，属于可选增强而非启动阻塞项

## 6. 常见启动异常快速排查

- [ ] 若数据库连接失败，优先检查 `docker compose ps`、`SPRING_DATASOURCE_URL`、schema 是否存在
- [ ] 若本地曾跑过旧版 `V1` 到 `V15` 迁移，先重建 `lattice` schema 或重建数据库，再启动当前版本
- [ ] 若 Redis 连接失败，优先检查 `LATTICE_REDIS_HOST`、`LATTICE_REDIS_PORT` 与容器健康状态
- [ ] 若 Maven 拉包失败，优先检查当前 `java` 版本与全局镜像是否异常，必要时改用 `.codex/maven-settings.xml`
- [ ] 若健康检查不通过，优先看 Spring Boot 启动日志中的 `Flyway`、`DataSource`、`Redis` 首个异常
- [ ] 若全文检索效果异常，先确认是否自动回退到了 `simple`，不要默认认为 `jiebacfg` 一定存在
- [ ] 若需要真实模型调用，再补充有效的 OpenAI / Anthropic API Key；当前本地基线不要求真实可用

## 7. 当前推荐启动口径

适合当前仓库的本地启动口径如下：

1. 使用 JDK 21。
2. 先拉起 `postgres` 与 `redis`。
3. 创建 `lattice` schema。
4. 显式导出数据库、Flyway、Redis 与占位 API Key 配置。
5. 优先用 `mvn -s .codex/maven-settings.xml spring-boot:run` 启动。
6. 以 `/actuator/health` 和 `\dt lattice.*` 作为最小验收。
